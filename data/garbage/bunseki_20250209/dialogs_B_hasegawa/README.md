# モデルB(対話17~32)対話の構造化データ

## データファイルの構成
- dialogs_B.html
  - モデルBの対話データ(対話17~32、橋本さんから頂いた生データ)
- dialogs_B_(ペルソナの名前).md (全16ファイル)
  - 各ペルソナ対話の構造化データ
  - 例: `dialogs_B_Endo.md`はEndoペルソナのモデルBの対話データ
  - 全16ファイル
  - インデントの深さで階層構造を表現しています。
- dialogs_B_(ペルソナの名前).html (全16ファイル)
  - dialogs_B_(ペルソナの名前).mdをVisual Studio Codeの拡張機能「Markmap」を用いて階層構造を図として示したデータ。
- dialog_B.md
  - 全16ペルソナの対話構造化データをマージした全体の構造化データ
- dialog_B.html
  - dialog_B.mdの階層構造を図として示したもの

## 構造化データの作成手順
1. 対話内容の分解
   - モデルBの対話データを読み、内容を構成要素ごとに細分化する。意味のまとまりや情報ごとのまとまりで分解した。
2. 階層構造の整理
   - 細分化した要素について、関係性（つながり・親子関係など文章の内容を読んで判断する）を考慮しながら階層構造の形で整理する。
3. 各ペルソナの分析
   - 全16ペルソナの対話データについて、1.2.のプロセスを行う。
4. 全16ペルソナのマージ
   - 16の構造化データを統合し、全体の構造化データを作成する（データを統合しやすくするために、1および2の段階で共通の要素名やフォーマットを意識しながら作業を進め、最後に統一された全体構造を完成させます。）
