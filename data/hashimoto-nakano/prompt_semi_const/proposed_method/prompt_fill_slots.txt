インタビュアーとインタビュー対象者である看護師の対話履歴を分析し、インタビュー対象者のペルソナ情報をスロットに追加します。
スロットにペルソナ情報を追加する際は、インタビュー対象者が回答で使用した言葉や用語を反映させます。
既に情報が入力されているスロットに情報を追加することもできます。

# スロットの説明
スロットを用いて看護師のペルソナ情報を整理し、記録します。
各スロットはスロット名、question_priority、value の3つの要素で構成されます。
- スロット名: ペルソナ情報を記録するスロットの名前です。
- question_priority: この項目は変更しないでください。
- value: スロットに記録するペルソナ情報です。
- 例: {{"現在のキャリア": {{"question_priority": 0, "value": "大学病院の内科で勤務"}},"悩みや不満点": {{"question_priority": 0, "value": "人間関係がうまくいかない"}}}}

# スロットへの情報追加方法
1. 既存のスロットと対話履歴を分析し、インタビュー対象者の回答内容のうち、スロットに記録されていない情報を特定する。
2. その情報を既存のスロットのうち、最も最適と思われるスロットに追加する。既存のスロットに追加できない場合は、新たにスロットを作成せず、最も近い意味を持つスロットに追加する。
3. スロットに追加されていない情報は、昔の対話であっても該当するスロットに追加する。
4. インタビュー対象者が「わかりません」という回答をした場合、質問に用いたスロットの value に「このスロットに類似するスロットを生成しないでください。」と入力する。
例: {{"現在のキャリア": {{"question_priority": 1, "value": "このスロットに類似するスロットを生成しないでください。"}},"悩みや不満点": {{"question_priority": 2, "value": "人間関係がうまくいかない"}}}}

# 出力指示
必ず JSON 形式で出力してください。
元のスロットをすべて出力し、いかなるスロットも削除しないでください。
新規のスロットは絶対に作成しないでください。
question_priority の値は絶対に変更しないでください。
value のみに情報を追加してください。

# 出力例
{{
  "スロット名1": {{ // スロット名
    "question_priority": integer, // ここの数字は絶対に変更しないでください
    "value": "ここに情報を入力、追加" // スロットに情報を入力
  }},
  "スロット名2": {{ // スロット名
    "question_priority": integer, // ここの数字は絶対に変更しないでください
    "value": null // 埋められない場合はnullのままにしてください
  }}
}}
