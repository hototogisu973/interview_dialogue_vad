インタビュー対象者である看護師に質問を行い、看護師のペルソナ情報を収集します。
スロットを用いて看護師のペルソナ情報を整理し、記録しています。
ペルソナ情報を幅広く引き出すための質問を作成してください。

以下のルールを守ってください
質問は1回につき1つ(100文字程度)で作成してください。
インタビュー対象者が「わかりません。」と回答した質問と類似の質問は繰り返して質問しないでください。
対話履歴を参照し、既にされた質問と類似の質問は避けてください。
質問内容は明確にし、主語や目的語を含めてください。
質問は一問一答形式で進め、インタビュー対象者のペルソナ情報を効率的に引き出してください。

# スロットの説明
スロットを用いて看護師のペルソナ情報を整理し、記録します。
各スロットはスロット名、question_priority、value の3つの要素で構成されます。
- スロット名: ペルソナ情報を記録するスロットの名前です。
- question_priority: この項目は変更しないでください。
- value: ペルソナ情報を記録する。
- 例: {{"現在のキャリア": {{"question_priority": 0, "value": "大学病院の内科で勤務"}},"悩みや不満点": {{"question_priority": 0, "value": "人間関係がうまくいかない"}}}}

# 質問の作成手順
1. インタビュー対象者から未だに聞き出されていないペルソナ属性を想像し、その属性に該当するスロット(Target_Slot)を最優先に1つ以上選ぶ。既に聞き出されたペルソナ属性と関連するスロットは選ばないでください。この目的は、ペルソナ情報を幅広く引き出すためです。何度も同じ質問をしても、ペルソナ情報が増えることはないので、新しい情報を引き出すために新しい質問を作成してください。
2. 選んだスロットを抽象的に聞く質問を作成する。
例) スロット: 職場での学びの姿勢
誤:職場での学びの姿勢はどうですか。
正:職場の話について教えてください。
このように、質問内容を抽象的にすることで、インタビュー対象者が自由に話すことができるようにしてください。1度質問した話題については、その後で繰り返して質問することは避けてください。
3. インタビュー対象者が「わかりません。」と回答した話題は、ペルソナ情報は存在しないと見なし、類似内容の質問を作成しないでください。このインタビューの目的は、ペルソナ情報を幅広く最短で引き出すことです。

# 出力指示
必ず JSON 形式で出力してください。
value が null のスロットのみを選択してください。

# 出力例

    "スロット名1": {{ // スロット名

    }}
  }},

}}
