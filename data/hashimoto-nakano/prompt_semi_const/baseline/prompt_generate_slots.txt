インタビュー対象者である看護師との対話履歴を分析し、看護師のペルソナ情報を収集するスロットを作成します。
対話履歴を見て、インタビュー対象者のペルソナ情報の整理に使う新しいスロットを1つだけ生成するか、新しいスロットを生成しないかを判断してください。
すでに存在するスロットと概念的に重複しないように注意しながら、効率的に情報を引き出せる抽象度の高いスロット名を生成してください。

# 新しいスロットの作成方法
1. 対話履歴の分析: 相談中に看護師が表明したペルソナ情報を抽出します。
2. スロットの生成: 抽象度の高いスロット名(抽象度が高いスロット名の例：「キャリア」「悩みや不満点」などキーワードで端的に表現したもの)を用いてください。
- 既存のスロットと同名もしくは類似のコンセプトのスロットは絶対に生成しないでください。
- valueに「このスロットに類似するスロットを生成しないでください。別の抽象的なスロットを作成してください。」と入力されているスロットはすでに情報がないと見なします。そのため、そのスロットと類似するスロットも絶対に生成しないでください。
3. 過去の対話履歴を振り返り、ペルソナ情報としてスロットに抽出されていない情報があれば、新しいスロットを生成してください。

# スロットの説明
スロットを用いて看護師のペルソナ情報を整理し、記録します。
各スロットはスロット名、question_priority、value の3つの要素で構成されます。
- スロット名: ペルソナ情報を記録するスロットの名前です。
- question_priority: 新しく作成するスロットは常に0に設定してください。
- value: 新しく作成するスロットは常にnullに設定してください。
例: {{"現在のキャリア": {{"question_priority": 0, "value": null}},"悩みや不満点": {{"question_priority": 0, "value": null}}}}
スロット名は以下を参考に、抽象度の高いスロット名を生成してください。
["性格","過去のキャリア","転職","SNS","趣味","家族","ライフスタイル","SNS","出身地","夢や目標","タスク管理","失敗談","育児","昇進","コミュニケーション","人間関係","ストレス","コミュニケーション","価値観","尊敬する人"],

# 注意事項
既存スロットと同名または類似コンセプトのスロットは生成しない。
「このスロットに類似するスロットを生成しないでください。」とvalueに設定されているスロットと類似のスロットを再度生成しない。

# 出力指示
必ず JSON 形式で出力してください。
新しいスロットを1つ生成するか、新しいスロットを生成しないかを判断してください。
出力には既存のスロットを含めないで、新たに作成したスロットのみを出力してください。
新しいスロットを生成する必要がないと判断した場合は `None` を出力してください。

# 出力例
{{
  "現在のキャリア": {{ // 既存のスロットと重複しない抽象的なスロット名
    "question_priority": 0, // 常に0に設定
    "value": null // 常にnullに設定
  }}
}}
